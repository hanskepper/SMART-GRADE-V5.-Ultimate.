## Reporting a Vulnerability

If you discover a security vulnerability in SMART GRADE, please report it responsibly.

### How to Report

1. **Email:** Send details to hanskepper52@gmail.com
2. **WhatsApp:** +237 698 640 885
3. **GitHub:** Open a private issue (mark as "security")

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any possible fixes (optional)

### Response Timeline

| Step | Timeline |
|------|----------|
| Acknowledgment | Within 24 hours |
| Initial Assessment | Within 3 days |
| Fix (if accepted) | Within 7-14 days |
| Public Disclosure | After fix is released |

### What to Expect

- ✅ **Accepted:** We will fix the issue and credit you in the release notes
- ❌ **Declined:** We will explain why and provide recommendations
- 🔒 **Confidential:** All reports are treated confidentially

## Security Features

SMART GRADE implements the following security measures:

| Feature | Description |
|---------|-------------|
| **Local Storage** | All data stored locally on user device |
| **PIN Protection** | 4-digit PIN for account access |
| **Fingerprint** | Optional biometric authentication |
| **No External Servers** | Data never leaves device by default |
| **Transfer Codes** | Expire after 10 minutes |
| **Auto-Cleanup** | Expired codes automatically deleted |

---

## Responsible Disclosure

We follow responsible disclosure practices:

1. Report vulnerability privately
2. Allow time for fix (7-14 days)
3. Release fix before public disclosure
4. Credit the reporter (if desired)
